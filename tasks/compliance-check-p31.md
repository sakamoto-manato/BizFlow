## 共P-04：会計・財務・経営 — OK

### 注意点への対応確認

PDF P.31 の共P-04 注意点：

> **「仕訳機能や会計ソフトへの連携機能がなく、経費費目と金額を入力・表示するだけのものは対象外とする。」**

BizFlowは **仕訳機能を実装済み** であり、単なる経費入力・表示ではない。以下の通り、複式簿記に基づく本格的な会計機能を搭載している。

### 4つのサブビュー構成

AccountingPageは PillFilter で切替可能な4つのサブビューで構成：

| サブビュー | 機能概要 | 該当箇所 |
|-----------|---------|---------|
| 入金概要 | 月別入金額・累積入金額を管理。エリアチャート・バーチャートで入金推移を可視化 | `AccountingPage` overview (App.jsx:541-573) |
| インボイス仕分け | 取引を適格請求書（T番号あり）と免税事業者（T番号なし）に自動仕分け。仕入税額控除の可否を判定・控除可能消費税額を算出 | App.jsx:576-650 |
| 仕訳帳 | 複式簿記による仕訳入力。借方科目・貸方科目・金額・摘要を入力し、伝票番号を自動採番（JE-YYYY-NNN） | App.jsx: journal ビュー |
| 試算表 | 残高試算表をカテゴリ別（資産/負債/純資産/収益/費用）に集計。貸借一致の自動検証機能付き | App.jsx: trial-balance ビュー |

### 仕訳機能（複式簿記）の実装詳細

| 要件 | 対応機能 | 該当箇所 |
|------|---------|---------|
| 仕訳帳入力 | 日付・借方科目・貸方科目・金額・摘要を入力する仕訳登録フォーム。借方/貸方を明確に分離した複式簿記形式 | App.jsx: journal ビュー内フォーム |
| 勘定科目マスタ | 5カテゴリ・18科目の勘定科目マスタ。ドロップダウンでカテゴリ別（資産/負債/純資産/収益/費用）にグループ表示 | `accounts` テーブル (schema.sql) |
| 伝票番号の自動採番 | JE-{YYYY}-{連番} 形式で自動生成（例: JE-2026-001） | App.jsx: entryNo 生成ロジック |
| 仕訳帳一覧 | 伝票No・日付・借方科目・貸方科目・金額・摘要を6列テーブルで表示 | App.jsx: journal テーブル |
| 月次フィルタ | 当月の仕訳件数・借方合計・貸方合計をメトリクスカードで表示 | App.jsx: monthEntries |

### 勘定科目マスタ（18科目）

| カテゴリ | 科目数 | 主な科目 |
|---------|-------|---------|
| 資産 | 4科目 | 現金(101)、普通預金(102)、売掛金(110)、商品(120) |
| 負債 | 3科目 | 買掛金(201)、未払金(202)、預り金(210) |
| 純資産 | 1科目 | 資本金(301) |
| 収益 | 2科目 | 売上高(401)、受取利息(410) |
| 費用 | 8科目 | 仕入高(501)、給与手当(510)、法定福利費(520)、旅費交通費(530)、通信費(540)、消耗品費(550)、支払手数料(560)、地代家賃(570) |

### 残高試算表の実装詳細

| 要件 | 対応機能 | 該当箇所 |
|------|---------|---------|
| 残高試算表 | 全勘定科目のコード・科目名・借方合計・貸方合計・残高を一覧表示 | App.jsx: trial-balance ビュー |
| カテゴリ別小計 | 資産/負債/純資産/収益/費用の5カテゴリごとに小計行を表示。色分けで視覚識別 | カテゴリ別 subtotal 行 |
| 貸借一致検証 | 借方合計と貸方合計の一致を自動検証。一致時は「貸借一致」（緑バッジ + CheckCircle2）、不一致時は「貸借不一致」（赤バッジ + AlertCircle）を表示 | grandDebit === grandCredit 判定 |
| 正常残高判定 | 資産・費用は借方残高、負債・純資産・収益は貸方残高として正しく計算 | isDebitNormal ロジック |

### データベース構成

```sql
-- 勘定科目マスタ
CREATE TABLE accounts (
  id bigint PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  code text UNIQUE NOT NULL,        -- 科目コード（例: "101"）
  name text NOT NULL,               -- 科目名（例: "現金"）
  category text NOT NULL,           -- 'asset'|'liability'|'equity'|'revenue'|'expense'
  sort_order integer DEFAULT 0
);

-- 仕訳帳
CREATE TABLE journal_entries (
  id bigint PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  entry_date date NOT NULL,                            -- 仕訳日付
  entry_no text UNIQUE NOT NULL,                       -- 伝票番号（JE-2026-001）
  description text DEFAULT '',                         -- 摘要
  debit_account_id bigint REFERENCES accounts NOT NULL,  -- 借方科目
  credit_account_id bigint REFERENCES accounts NOT NULL, -- 貸方科目
  amount bigint NOT NULL,                              -- 金額
  created_at timestamptz DEFAULT now()
);
```

### 注意点条件の充足根拠

「仕訳機能や会計ソフトへの連携機能がなく、経費費目と金額を入力・表示するだけのものは対象外」に対して：

1. **仕訳機能あり** — 借方/貸方を明確に分離した複式簿記形式の仕訳入力フォームを実装
2. **単なる金額表示ではない** — 勘定科目マスタ（18科目）に基づく仕訳帳、残高試算表、貸借一致検証機能を搭載
3. **インボイス仕分け** — T番号に基づく適格/免税の自動分類と仕入税額控除可否判定を実装

## まとめ

- **全5プロセス（共P-01〜共P-05）が準拠**しており、通常枠（4種類以上）の要件を充足。
- **共P-04 の注意点（「仕訳機能がなく経費費目と金額を入力・表示するだけのものは対象外」）については、複式簿記の仕訳帳・勘定科目マスタ（18科目）・残高試算表（貸借一致検証付き）を実装済み**であり、単なる金額入力・表示ではないことを明確に示せる。
- **共P-01 は取引先管理・T番号管理・適格/免税バッジ・検索/フィルタ/ソート機能**を搭載し、顧客対応・販売支援プロセスとして準拠。
- インボイス枠での申請も、「会計」機能（4要件）+「受発注」機能（6要件）の両方を充足しているため可能。
